# 演示脚本

本演示用于说明 SupportPilot AI 的 **通用客服运营平台** 能力。当前问题样例来自默认 `demo_ecommerce` 电商售后 Demo 模板，仅用于展示平台如何完成知识检索、风险判断、转人工和建工单闭环。

## 演示前准备

1. 启动后端与前端。
2. 打开前端工作台。
3. 进入“流程试运行”区域。
4. 确认后端使用默认配置：

```env
LLM_PROVIDER=mock
RAG_RETRIEVER=hybrid
RAG_QUERY_REWRITE=true
SUPPORTPILOT_TEMPLATE=demo_ecommerce
```

## 场景 1：标准问题自动回复

客户选择：`shop_001`

输入：

```text
我的快递什么时候发货？
```

讲解点：

- 这是电商售后 Demo 中的标准履约进度问题。
- 平台执行通用流程：意图识别 → 风险判断 → 知识库检索 → 生成回复 → 答复校验 → 动作决策。
- RAG 命中物流 / 履约相关知识片段。
- AI 输出带引用回答。
- 最终 action 为 `auto_reply`。
- 该能力可迁移到其他行业，例如 SaaS 的“服务申请进度”、教育的“课程安排查询”、本地生活的“预约状态查询”。

## 场景 2：政策类问题自动回复

客户选择：`shop_003`

输入：

```text
我想退货，七天无理由怎么申请？
```

讲解点：

- 这是电商售后 Demo 中的政策咨询问题。
- 平台通过知识库检索返回规则依据，而不是直接编造答案。
- 如果知识库有明确规则，AI 自动回复并展示 citation。
- 最终 action 通常为 `auto_reply`。
- 在真实业务中，企业可上传自己的退款、取消、改签、SLA、合同或服务政策文档。

## 场景 3：异常交付自动建单

客户选择：`shop_001`

输入：

```text
收到的杯子碎了，外包装也变形了
```

讲解点：

- 这是电商售后 Demo 中的异常交付问题。
- 平台识别为需要持续跟进的服务异常。
- 风险判断标记为需要创建服务工单。
- 系统调用 mock 工具创建工单，并发送 mock 通知。
- 最终 action 为 `create_ticket`。
- 在通用平台中，这类能力可对应 SaaS 故障、服务交付异常、线下履约问题、材料审核异常等场景。

## 场景 4：投诉 / 赔偿转人工

客户选择：`shop_003`

输入：

```text
我要投诉你们并要求赔偿
```

讲解点：

- 这是高风险请求示例。
- 平台识别到投诉、赔偿等升级风险。
- AI 不直接承诺赔偿或处理结果。
- 系统生成转人工说明，并将请求进入人工队列。
- 最终 action 为 `handoff`。
- 在真实业务中，高风险规则可扩展为隐私、安全、合同、舆情、重大故障、VIP 客户等升级策略。

## 知识库导入演示

可准备一篇 Markdown 文档，例如：

```markdown
# 服务响应规范

## 标准响应时效

普通服务请求应在 24 小时内响应，高风险请求应优先进入人工队列。

## 升级处理

涉及投诉、赔偿、隐私、安全的问题必须由人工专员复核。
```

演示步骤：

1. 进入“知识库”模块。
2. 点击“导入 Markdown”。
3. 选择 `.md` 文件。
4. 导入成功后点击“重建索引”。
5. 在流程试运行中输入与文档相关的问题，观察 citation 是否引用新文档。

讲解点：

- 行业知识不需要写死在后端。
- 企业可以通过工作台上传自己的 SOP、FAQ、政策和服务规则。
- 电商售后只是默认 Demo 模板，真实客户可替换为自己的知识库。

## 演示总结

SupportPilot AI 的核心价值不是“回答某个固定行业的问题”，而是提供一套通用客服运营闭环：

```text
客户请求
  → AI 工作流
  → 知识库检索
  → 风险判断
  → 自动回复 / 转人工 / 建工单
  → 处理记录
  → 指标复盘
  → 知识库持续更新
```

后续可继续扩展：

- 多行业模板
- 模板知识库一键导入
- 可配置 intent set
- PDF / DOCX / 企业文档导入
- 多租户、权限和真实客服渠道接入
